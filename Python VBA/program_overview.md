# extract_from_filenames.py 処理概要

## 何をするプログラムか

`dummy_images/` フォルダ内の画像ファイル名を解析し、**測定値**と**破断状態**を自動抽出して `extraction_report.md` に出力する。

---

## ファイル名の構造

```
84-50TE04①_7.56um破断状態A.jpg
│    ││  ││ │    │       │
│    ││  ││ │    │       └─ .jpg（拡張子）
│    ││  ││ │    └─────────── 破断状態（um〜.jpg 間）← 抽出対象②
│    ││  ││ └──────────────── 測定値（_〜um 間）    ← 抽出対象①
│    ││  │└─────────────────── 破断方向（①②③）
│    ││  └──────────────────── サンプル番号（01〜05）
│    │└─────────────────────── 取得位置（TE / BM / RE など）
└────────────────────────────── ウェハ型番略称（84-50 など）
```

---

## 処理フロー

```
開始
 │
 ▼
dummy_images/ フォルダを読み込む
 │
 ▼
.jpg ファイル一覧をアルファベット順にソート
 │
 ▼
┌─────────────────────────────┐
│ 各ファイル名に対してループ    │
│                             │
│  正規表現でパターンマッチ     │
│  _([数値])um([文字]).jpg     │
│         │                   │
│    マッチ？                  │
│   Yes ／  No                │
│    ↓       ↓                │
│  辞書に   [SKIP] と         │
│  追加     コンソール表示      │
└─────────────────────────────┘
 │
 ▼
コンソールに結果を表示
（測定値・破断状態の一覧 + 成功件数）
 │
 ▼
extraction_report.md を生成
（サマリー表 ＋ 抽出データ一覧 ＋ 失敗ファイル一覧）
 │
 ▼
終了
```

---

## 正規表現の詳細

```python
pattern = r"_([0-9]+\.[0-9]+)um(.+?)\.jpg$"
```

| パーツ | 意味 | 例 |
|--------|------|----|
| `_` | アンダースコアを起点にする | `_` |
| `([0-9]+\.[0-9]+)` | 整数.小数 の数値を**グループ①**として取得 | `7.56` |
| `um` | `um` をセパレータとして読み飛ばす | `um` |
| `(.+?)` | 1文字以上（最短）を**グループ②**として取得 | `破断状態A` |
| `\.jpg$` | `.jpg` で終わることを確認 | `.jpg` |

### マッチ例

```
84-50TE04①_7.56um破断状態A.jpg
            ^^^^    ^^^^^^^^^
         group(1)  group(2)
          → 7.56   → 破断状態A
```

## 関数の役割

```
extract_from_filenames.py
├── extract_info(filename)   ← ファイル名 1件を解析 → dict または None
├── main()                   ← フォルダ読み込み・ループ・コンソール表示
└── write_report(results, skipped_files)  ← MDファイル生成
```

### `extract_info()` の入出力

```
入力: "84-12BM01②_5.34um破断状態B.jpg"

出力: {
    "filename"      : "84-12BM01②_5.34um破断状態B.jpg",
    "measurement"   : 5.34,          # float型
    "fracture_state": "破断状態B"
}

抽出失敗時: None
```

---

## 出力ファイル

| ファイル名 | 内容 |
|------------|------|
| `extraction_report.md` | 実行ごとに上書き生成。サマリー・抽出データ一覧・失敗ファイル一覧を含む |

---

## 実行方法

```bash
python3 extract_from_filenames.py
```
