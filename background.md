# 業務背景：半導体メモリ品質管理工程における検査業務

## 勤務先・担当業務

半導体メモリメーカーへの技術派遣として、品質管理部門の検査工程に従事。
マイクロスコープを使ったサンプル撮影と、TIBCO Spotfire（DXP）によるデータ集計・可視化レポートの作成が主な業務。

---

## 業務①：破断面観察

### 目的

半導体チップは製造後、ロボットアームに取り付けられたエアピンセットによってウェハ（円盤状の基板）から1枚ずつ取り出される。
このとき**チップの強度が不足していると、取り出し時の圧力でチップが割れ、不良品となる**。

この不良を減らすために、以下の2点を定量的に把握することが求められる：

- **ウェハ上のどの位置**のチップが強度不足になりやすいか
- **どの製造条件**で強度の低いチップが生まれやすいか

これらの傾向を分析して製造工程を改善し、**歩留まり（良品率）を向上させる**ことがこの観察の目的。

### 観察の流れ

1. サンプルチップに対して意図的に荷重をかけて破断し、破断時の圧力を記録する
2. 破断後の断面をマイクロスコープで撮影する
3. 撮影画像をファイル名に必要情報を埋め込んだ命名規則で保存する
4. Excel マクロでファイル名から各データを自動抽出し、管理台帳に転記する
5. TIBCO Spotfire でデータを集計・可視化し、強度のばらつきや傾向を分析・報告する

### 記録する項目

| 項目 | 内容 | 目的 |
|------|------|------|
| ウェハ型番 | 製造ロットの識別子 | 製造条件との紐付け |
| 取得位置 | ウェハ上の場所（Top Edge / Center など） | 位置による強度差の分析 |
| サンプル番号 | 同一条件内の個体識別 | データのばらつき把握 |
| 破断方向 | 荷重をかけた方向（3パターン） | 方向による強度差の分析 |
| 測定値 | 破断時の圧力（数値） | 強度の定量評価 |
| 破断状態 | 割れの起点となった箇所の分類 | 不良原因の特定 |

## 業務②：ALカットバリ確認

### 目的

ウェハからチップを切り出す際、専用のブレードでチップとチップの間の切り代（切削ライン）を切断する。
この切削ラインにはチップ間をつなぐアルミ製の接続部があり、ブレードが通過する際に**アルミがマイクロメートル単位の削りカス（バリ）としてチップ端面に残ることがある**。

バリは肉眼では確認できないサイズだが、**ボンディングワイヤー（チップと基板を接続する髪の毛よりも細い電線）に触れると、通電時に発熱しショートによる配線不良を引き起こす**。

これを未然に防ぐため、切り出し後のチップ端面をマイクロスコープで撮影し、バリの有無・形状・発生位置を記録する。

### 観察の流れ

1. ウェハをブレードで切り出し、個別のチップに分割する
2. 切り出し後のチップ端面をマイクロスコープで撮影する
3. バリの有無・形状をファイル名に情報を埋め込んで保存する
4. Excel マクロでデータを自動抽出し、管理台帳に転記する
5. TIBCO Spotfire でバリの発生傾向（位置・条件・頻度）を集計・分析する

---

## Excel マクロによる自動化が必要になった背景

上記2つの業務はいずれも、1回の作業で数十件の画像を撮影し、1,000行超の管理台帳にデータを転記する必要がある。

手作業では「1,000行の中から該当行を目視で探して1件ずつ入力する」という工程が避けられず、転記ミスが頻発していた。
ミスの修正と再確認のループが繰り返され、**1回の作業に3〜4時間**かかることがあった。

この課題に対し、ファイル保存時の命名規則を設計し直すことで、ファイル名そのものをデータソースとして活用できる構造を作った。
Excel 関数（MID・FIND・IFERROR・INDEX/MATCH）と VBA を組み合わせて自動抽出・自動転記の仕組みを構築した結果、**同じ作業を約45分で完了できるようになった**。
