# Excel 関数の設計詳細

## ファイル名の構造

```
84-50TE04①_X.XXum破断状態A.jpg
│    ││  ││ │    │
│    ││  ││ │    └── 破断状態（um〜.jpg の間）
│    ││  ││ └─────── 測定値（_〜um の間）
│    ││  │└────────── 破断方向（10文字目 ①②③等）
│    ││  └─────────── サンプル番号（8-9文字目）
│    │└──────────────── 取得位置（6-7文字目）
└────────────────────── ウェハ型番略称（1-5文字目）
```

---

## 各列の関数設計

### B列: 正式型番（IF ネストで変換）

```excel
=IF(MID(A2,1,5)="80-50","DH260507580-50",
 IF(MID(A2,1,5)="84-12","CE260402384-12",
  IF(MID(A2,1,5)="84-50","DCH003484-50","該当なし")))
```

**処理の流れ:**

1. `MID(A2, 1, 5)` → 先頭 5 文字の型番略称を抽出
2. `IF` のネストで略称→正式型番を照合
3. どの条件にも一致しなければ「該当なし」

| 略称 | 正式型番 |
|------|---------|
| 80-50 | DH260507580-50 |
| 84-12 | CE260402384-12 |
| 84-50 | DCH003484-50 |

**なぜ IF ネストを使ったか:**

- 当初は `LET + IFS` で実装したが、旧バージョンの Excel で `@` 演算子が自動挿入され `#NAME?` エラーが発生
- `IF` ネストなら**全バージョンの Excel で動作**する
- 型番マスタ用の別シートも不要で、1 つの数式内で完結する

### C列: 取得位置（IF ネストで変換）

```excel
=IF(MID(A2,6,2)="TE","Top Edge",
 IF(MID(A2,6,2)="BM","Bottom Middle",
  IF(MID(A2,6,2)="Ct","Center",
   IF(MID(A2,6,2)="LE","Left Edge",
    IF(MID(A2,6,2)="RE","Right Edge","該当なし")))))
```

`Edge`（外側）・`Middle`（中央）・`Center`（真ん中）と、Top / Right / Bottom / Left の 4 方向を組み合わせて場所を判断。

| コード | 変換結果 |
|--------|---------|
| `TE` | Top Edge（上・外側） |
| `BM` | Bottom Middle（下・中央） |
| `LE` | Left Edge（左・外側） |
| `RE` | Right Edge（右・外側） |
| `Ct` | Center（ど真ん中。範囲が狭いため 4 方向の区分なし） |

### D列: サンプル番号（数値変換）

```excel
=MID(A2, 8, 2) * 1
```

`MID` で抽出した文字列 `"04"` を `* 1` で数値 `4` に変換。Spotfire で数値型として集計できるようにする。

### E列: 破断方向 ― IF + IFERROR + FIND

```excel
=IF(IFERROR(FIND("①",A2),0)
   +IFERROR(FIND("②",A2),0)
   +IFERROR(FIND("③",A2),0)=0,
  "該当なし",
  MID(A2,
    IFERROR(FIND("①",A2),0)
   +IFERROR(FIND("②",A2),0)
   +IFERROR(FIND("③",A2),0),1))
```

**なぜ単純な `MID(A2, 10, 1)` を使わなかったのか:**

`MID` で固定位置から取得すると、ファイル名のフォーマットがわずかでもずれた場合に間違った値を返す。  
`IFERROR(FIND(...), 0)` の組み合わせなら、該当パターンが見つかった位置を正確に返し、見つからなければ 0 を返すため、想定外の入力にも耐えられる。  
合計が 0（どのパターンにも該当しない）の場合は「該当なし」を返す。

### F列: 測定値

```excel
=MID(A2, FIND("_", A2) + 1, FIND("um", A2) - FIND("_", A2) - 1) * 1
```

処理の流れ:

1. `FIND("_", A2)` → `_` の位置（例: 10）
2. `FIND("um", A2)` → `um` の位置（例: 14）
3. `MID(A2, 11, 3)` → `_` の次から `um` の前まで抽出
4. `* 1` → 文字列 "X.XX" を数値 X.XX に変換

`* 1` はテキスト→数値変換のテクニック。`VALUE()` でも同等だが、`* 1` の方が短く書ける。

### G列: 破断状態

```excel
=MID(A2, FIND("um", A2) + 2, FIND(".jpg", A2) - FIND("um", A2) - 2)
```

`um` と `.jpg` の間にある日本語テキストを抽出。破断がチップのどの部分から起きたかを示す分類データで、Spotfire でのフィルターとして使用。

### H列: 複合キー（まとめDB との照合用）

```excel
=B2&C2&D2&E2
```

正式型番・取得位置・サンプル番号・破断方向を結合した一意キー。まとめDB シートの MATCH 関数がこの列を検索して、正しい行に自動転記する。

## まとめDB シートの関数（自動転記）

このシートには約 1,000 行分の識別情報（正式型番・取得位置・No・破断方向）があらかじめ登録されており、E・F 列が貼り付け場所変換シートの該当行から値を自動取得する。

### E列: 測定値（INDEX + MATCH で自動転記）

```excel
=IFERROR(INDEX(貼り付け場所変換!F:F, MATCH(A2&B2&C2&D2, 貼り付け場所変換!H:H, 0)), "該当なし")
```

| ステップ | 処理内容 |
|---------|---------|
| `A2&B2&C2&D2` | まとめDB の型番・位置・番号・方向を結合して複合キーを作成 |
| `MATCH(..., 貼り付け場所変換!H:H, 0)` | 貼り付け場所変換の H 列（複合キー）を完全一致検索し、行番号を取得 |
| `INDEX(貼り付け場所変換!F:F, ...)` | 取得した行番号で F 列（測定値）の値を返す |
| `IFERROR(..., "該当なし")` | 未撮影サンプルの行はエラーでなく「該当なし」を表示 |

### F列: 破断状態（INDEX + MATCH で自動転記）

```excel
=IFERROR(INDEX(貼り付け場所変換!G:G, MATCH(A2&B2&C2&D2, 貼り付け場所変換!H:H, 0)), "該当なし")
```

E 列と同じロジックで G 列（破断状態）を返す。

**なぜ VLOOKUP を使わなかったか:**

VLOOKUP は検索列が必ず左端でなければならず、複数列のキー結合にも対応しにくい。
`INDEX + MATCH` の組み合わせなら検索列の位置を問わず、かつ複合キー（`A&B&C&D`）による多条件一致が 1 つの数式で実現できる。
